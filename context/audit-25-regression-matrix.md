# Audit Section 25 Regression Matrix

This manifest maps the P0/P1 findings in the authoritative 2026-07-16 deep
audit to behavior-specific regression coverage. Sections 01–09 and 11–15 are
in scope; Section 10 is P2 and is not part of the completion gate. The final
table records additional P2 tests that Section 25 explicitly asked to verify.

Database integration suites use `describe.skip` unless the database name from
`DATABASE_URL` contains `test`, and each guarded suite asserts that condition
again in `beforeAll`. They must never be made runnable against a differently
named database.

## Validation order

CI runs canonical workspace verification and install, Prisma client generation,
lint, formatting, typecheck, build, client plus server non-database tests,
migrations against CI's fixed `stafflow_test` environment, guarded database
integration tests, and database smoke checks. The raw Prisma migration command
has no database-name guard of its own. Smoke does not rebuild or repeat
typechecking.

The direct `@vitest/coverage-v8` dependency was intentionally removed in Audit
Section 21. Vitest 4.1.6 still installs a transitive copy, but a clean-install
coverage invocation fails because that copy cannot resolve `magicast`. No
coverage command or arbitrary threshold is added in Section 25; installing
extra packages solely to repair optional visibility would contradict the audit
instruction.
